# Bank Accounts OOP

Навчальний Java-проєкт для роботи з банківськими рахунками.

Проєкт демонструє використання принципів ООП, власних виключень, шаблонів проєктування, історії транзакцій і автоматизованого тестування через Maven та JUnit.

## Тема проєкту

Робота з банківськими рахунками.

## Основні можливості

Програма дозволяє:

- створювати різні типи банківських рахунків;
- поповнювати рахунок;
- знімати кошти;
- виконувати переказ між рахунками;
- враховувати комісію під час переказу;
- зберігати історію транзакцій;
- обробляти помилки банківських операцій.

## Типи рахунків

У проєкті реалізовано такі типи рахунків:

- `DebitAccount` — дебетовий рахунок;
- `CreditAccount` — кредитний рахунок з кредитним лімітом;
- `SavingsAccount` — накопичувальний рахунок з відсотковою ставкою.

Базовим класом для них є абстрактний клас `Account`.

## Структура проєкту

```text
src/main/java/org/example/bank
├── command      командний шаблон для операцій
├── exception    власні виключення
├── facade       спрощений інтерфейс роботи з системою
├── factory      створення рахунків
├── model        основні моделі предметної області
├── observer     сповіщення про транзакції
├── service      логіка роботи з рахунками
├── strategy     стратегії розрахунку комісії
├── system       Singleton для банківської системи
└── Main.java    демонстрація роботи програми
```

## Використані шаблони проєктування

У роботі використано такі шаблони:

- `Factory` — створення різних типів рахунків;
- `Builder` — поступове створення об’єкта з даними рахунку;
- `Singleton` — єдиний об’єкт банківської системи;
- `Strategy` — різні способи розрахунку комісії;
- `Command` — банківські операції як окремі команди;
- `Observer` — сповіщення після виконання транзакції;
- `Facade` — спрощений інтерфейс для роботи з банком.

## Тестування

Для тестування використано Maven та JUnit 5.

Запуск тестів:

```powershell
mvn test
```

У проєкті перевіряються:

- поповнення рахунку;
- зняття коштів;
- переказ з комісією;
- помилка при недостатній кількості коштів;
- помилка при неправильній сумі;
- помилка при відсутньому рахунку;
- робота кредитного ліміту.

## Запуск програми

Спочатку зібрати проєкт:

```powershell
mvn compile
```

Запустити демонстрацію:

```powershell
java -cp target/classes org.example.bank.Main
```
